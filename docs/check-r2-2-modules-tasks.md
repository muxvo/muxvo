## 问题清单

### 高优先级

1. **更新检测模块归属不够明确**
   - 问题：PRD 功能 T（更新检测与推送）在 DEV-PLAN 中有独立的任务拆分（T-1 至 T-4），但没有独立模块。skill-browser (4.7) 的职责描述中未显式提及"更新检测"职责，仅通过"6.14 + 6.15 状态机"间接暗示。
   - 建议：在 skill-browser (4.7) 的职责描述中增加"已安装包更新检测与推送"的显式描述，或在 §4 中明确 T-1~T-4 属于 skill-browser 模块。

2. **PRD §8.4 Plugins 浏览功能的任务拆分缺失**
   - 问题：PRD §8.4 配置管理器列出了 9 种资源类型（Skills/Hooks/Plans/Tasks/Settings/CLAUDE.md/Memory/MCP/**Plugins**），DEV-PLAN §5 目录结构中有 `components/plugins/PluginList.tsx` 和 `PluginDetail.tsx`，但 §9 任务拆分中的 K1-K5 仅覆盖 Skills/Plans/Hooks/Tasks/Memory+MCP，没有 Plugins 的独立任务。
   - 影响：Plugin 浏览 UI 可能被遗漏。
   - 建议：在 K5 中补充"Plugins 只读浏览"或新增 K6 任务。

### 中优先级

3. **V2-P3 社区平台无模块和任务拆分**
   - 问题：PRD 功能 SC（Showcase 社区平台）在 DEV-PLAN §14 仅以概要形式描述（后端服务、Feed 流、排行榜等），没有模块定义和任务拆分。
   - 影响：无实际影响，V2-P3 明确定义了启动前提条件（月活>1000, 展示页月生成>100, 外部链接月点击>500），待条件满足后再详细规划。
   - 建议：当前状态可接受，但建议在 DEV-PLAN 中标注"V2-P3 详细模块和任务拆分将在启动前提满足后编写"。

4. **Review 数据结构(7.4)无对应实现任务**
   - 问题：PRD §7.4 定义了 Review（评价）数据结构，但 DEV-PLAN 明确说明此数据结构归属 V2-P3 社区平台。
   - 影响：无实际影响，DEV-PLAN §4.12 已做明确说明。
   - 建议：当前状态可接受。

5. **Skill 使用分析（PRD §3.10）的任务归属分散**
   - 问题：PRD §3.10 定义了"使用分析增强"（追踪调用次数、使用会话数、最后使用时间等），数据结构在 marketplace.json 的 `skillUsage` 字段。DEV-PLAN 中 analytics (4.12) 负责事件采集，O-3 负责 marketplace.json，但没有明确的任务专门负责 skillUsage 数据的采集和 UI 展示。
   - 建议：在 X8（使用统计面板）或 O-3 任务中补充 skillUsage 数据采集和展示的说明。

6. **AI 评分中使用数据授权弹窗缺少独立任务**
   - 问题：PRD §8.9 提到评分时可选纳入使用数据（`includeAnalytics` 参数），需用户授权弹窗确认。DEV-PLAN SR-3 的任务描述中未明确包含此授权 UI。
