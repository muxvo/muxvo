## E. 排期合理性

### E1. V1 各阶段工期分配

| 阶段 | 预估工期 | 功能任务天数合计 | 缓冲时间 | 缓冲比例 | 合理性评估 |
|---|---|---|---|---|---|
| V1-P0 | 6-7 周（30-35 工作日） | A(10.5d) + B(3d) + I(3d) + D(8.5d) + G(3d) = 28d | 2-7d | 7%-25% | M2 说明含 Bug 修复 1.5d + 联调 1.5d，但 M1(W1-3) 的 28d 紧凑 |
| V1-P1 | 5-6 周（25-30 工作日） | C(4d) + E(6d) + H(6d) + J(3d) + M(2d) + RE1(8d) = 29d | -4d ~ +1d | **不足** | 29d 任务挤入 25-30d，缓冲不足 |
| V1-P2 | 6-7 周（30-35 工作日） | F(3d) + K(4d) + L(3d) + RE2(5d) + X(15.5d) = 30.5d | -0.5d ~ +4.5d | 0%-15% | 含 X 系列基础设施任务，缓冲略紧 |
| **V1 合计** | **17-20 周** | **87.5d** | **-2.5d ~ +12.5d** | | 下限 17 周 = 85d，几乎无缓冲 |

**评估**：
- V1-P1 最为紧张：29d 功能任务挤入 25-30d，取下限时任务溢出。建议将 V1-P1 上调至 6-7 周。
- V1 合计下限 17 周 vs 87.5d 任务量，几乎无缓冲余量，风险偏高。
