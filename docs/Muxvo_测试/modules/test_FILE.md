       │                ┌────────────┐  自动重试(3s, max 3)  │

       │                       │
       │                  重试失败(3次)

## 4. L2 用例（场景与边界验证）

### 4.1 FILE_L2_001_三种关闭方式一致性

- **优先级**：P0
- **前置条件**：文件面板已打开
- **测试步骤**：
  1. 点击 ✕ 关闭面板 → 记录关闭动画和最终状态
  2. 重新打开面板
  3. 按 Esc 关闭面板 → 记录关闭动画和最终状态
  4. 重新打开面板
  5. 点击面板外区域关闭 → 记录关闭动画和最终状态
- **预期结果**：
  - 三种方式关闭动画一致
  - 最终状态均为 Closed
  - 面板宽度归零，不残留可见元素

### 4.2 FILE_L2_002_终端切换无动画

- **优先级**：P1
- **前置条件**：面板已打开，显示终端 A 的文件
- **测试步骤**：
  1. 点击终端 B 的文件按钮
  2. 观察面板行为
  3. 快速在终端 A 和终端 B 之间切换文件按钮（连续点击 5 次）
- **预期结果**：
  - 步骤 1-2：面板内容直接切换，无滑入滑出动画
  - 步骤 3：内容正确切换到最后点击的终端，无动画残影/闪烁

### 4.3 FILE_L2_003_拖拽边界值

- **优先级**：P1
- **前置条件**：三栏临时视图已进入
- **测试步骤**：
  1. 拖拽左栏边界，尝试缩小到 100px（< min 150px）
  2. 拖拽左栏边界，尝试扩大到 600px（> max 500px）
  3. 拖拽右栏边界，尝试缩小到 100px
  4. 拖拽右栏边界，尝试扩大到 600px
  5. 同时将左右栏都设为 500px
- **预期结果**：
  - 步骤 1：左栏宽度停在 150px，不可更小
  - 步骤 2：左栏宽度停在 500px，不可更大
  - 步骤 3-4：右栏同理，限制在 150px ~ 500px
  - 步骤 5：中栏 flex 区域被压缩，但仍有最小可用空间

### 4.4 FILE_L2_004_Markdown编辑保存流程

- **优先级**：P0
- **前置条件**：临时视图中打开一个 .md 文件，处于 PreviewMode
- **测试步骤**：
  1. 按 ⌘/ 进入 EditMode
  2. 修改 Markdown 内容（添加一行文本）
  3. 按 ⌘S 保存
  4. 按 ⌘/ 切换回 PreviewMode
  5. 验证渲染结果包含新增内容
  6. 用外部编辑器打开同一文件
- **预期结果**：
  - 步骤 3：保存成功，状态 Editing -> Saving -> Editing
  - 步骤 4：切换回预览模式（无未保存提示，因已保存）
  - 步骤 5：渲染结果包含新增文本
  - 步骤 6：外部编辑器中文件内容已更新

### 4.5 FILE_L2_005_未保存修改切换提示

- **优先级**：P0
- **前置条件**：临时视图 EditMode，已修改内容但未保存
- **测试步骤**：
  1. 在 EditMode 中修改文件内容
  2. 不保存，直接按 ⌘/ 尝试切换到 PreviewMode
  3. 观察提示对话框
  4. 选择「取消」→ 留在 EditMode
  5. 再次按 ⌘/ → 弹出提示
  6. 选择「放弃修改」→ 切换到 PreviewMode
  7. 验证文件内容恢复原状
- **预期结果**：
  - 步骤 2-3：弹出 UnsavedPrompt 提示
  - 步骤 4：留在 EditMode，修改内容保留
  - 步骤 6：切换到 PreviewMode
  - 步骤 7：预览内容为保存前的原始内容

### 4.6 FILE_L2_006_左栏终端数量边界

- **优先级**：P1
- **前置条件**：同一目录下有 5 个终端
- **测试步骤**：
  1. 进入三栏临时视图
  2. 观察左栏终端列表
  3. 滚动左栏查看所有终端
  4. 点击不同的终端，观察中栏变化
- **预期结果**：
  - 最多 3 个终端可见，其余需滚动
  - 滚动流畅
  - 点击不同终端，中栏显示对应文件

### 4.7 FILE_L2_007_文件监听错误恢复

- **优先级**：P1
- **前置条件**：文件面板已打开，监听中
- **测试步骤**：
  1. 模拟监听错误（如临时删除被监听目录）
  2. 等待 3 秒，观察第一次自动重试
  3. 再次模拟错误
  4. 等待 3 秒，观察第二次重试
  5. 恢复目录
  6. 等待 3 秒，观察第三次重试
- **预期结果**：
  - 状态进入 WatchError
  - 每 3 秒自动重试一次
  - 步骤 6：第三次重试成功，恢复 Watching 状态
  - 文件列表正确刷新

### 4.8 FILE_L2_008_文件监听重试耗尽

- **优先级**：P1
- **前置条件**：文件面板已打开，监听中
- **测试步骤**：
  1. 模拟持续监听错误（删除目录且不恢复）
  2. 等待 3 次自动重试（共约 9-12 秒）
  3. 观察最终状态
  4. 观察界面提示
- **预期结果**：
  - 3 次重试均失败
  - 状态最终进入 Inactive
  - 界面显示友好错误提示，告知用户文件监听已停止

### 4.9 FILE_L2_009_非Markdown文件处理

- **优先级**：P1
- **前置条件**：临时视图已进入
- **测试步骤**：
  1. 选择一个 .js 文件
  2. 观察中栏显示
  3. 尝试按 ⌘/ 切换模式
  4. 选择一个 .png 图片文件
  5. 选择一个二进制文件
- **预期结果**：
  - 步骤 2：代码文件以源码形式显示（语法高亮）
  - 步骤 3：非 Markdown 文件的模式切换行为合理（可能仅有源码模式）
  - 步骤 4：图片文件正确预览
  - 步骤 5：二进制文件提示不可预览

### 4.10 FILE_L2_010_外部修改冲突

- **优先级**：P1
- **前置条件**：临时视图 EditMode 正在编辑某文件
- **测试步骤**：
  1. 在 EditMode 中修改文件内容
  2. 使用外部编辑器同时修改同一文件并保存
  3. 观察 Muxvo 的反应
  4. 尝试在 Muxvo 中保存（⌘S）
- **预期结果**：
  - chokidar 检测到外部修改
  - Muxvo 提示文件已在外部被修改
  - 用户可选择：使用本地版本覆盖 / 放弃本地修改加载外部版本
