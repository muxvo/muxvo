## 1. 模块架构概览

```
  ┌──────────────────────────────────────────────────────────────────────┐
  │                         DATA 模块                                   │
  │                                                                     │
  │  ┌─────────────┐    ┌──────────────┐    ┌──────────────────────┐    │
  │  │  CC 数据     │    │ Muxvo 镜像    │    │  搜索索引              │    │
  │  │  (只读)      │    │ (读写)        │    │  (倒排索引)            │    │
  │  └──────┬──────┘    └──────┬───────┘    └──────────┬───────────┘    │
  │         │                  │                       │                │
  │         ▼                  ▼                       ▼                │
  │  ┌─────────────────────────────────────────────────────────────┐    │
  │  │              chokidar 文件监听层                               │    │
  │  │  ignored: [/\.DS_Store$/, /\.swp$/, /~$/]                   │    │
  │  └─────────────────────────────────────────────────────────────┘    │
  │         │                  │                       │                │
  │         ▼                  ▼                       ▼                │
  │  ┌──────────┐    ┌────────────────┐    ┌───────────────────┐       │
  │  │ JSONL    │    │ sync-state.json │    │ search-index/     │       │
  │  │ 解析器    │    │ (同步状态记录)   │    │ (持久化索引)       │       │
  │  └──────────┘    └────────────────┘    └───────────────────┘       │
  └──────────────────────────────────────────────────────────────────────┘

  同步流程：
  启动 → 全量扫描 → 建立 chokidar 监听 → 增量同步 → 按 sessionId 去重
  规则：仅同步不删除（CC 删了，Muxvo 保留）
```

## 2. L1 用例（功能验证）

### 2.1 CC 数据读取

| 用例编号 | 优先级 | 场景 | 预期结果 |
|---------|--------|------|---------|
| DATA_L1_001_只读访问 | P0 | 读取 ~/.claude/ 下的数据文件 | 以只读方式访问，不修改 CC 原始数据 |
| DATA_L1_002_未知字段兼容 | P0 | CC 数据中包含 Muxvo 未定义的字段 | 未知字段被忽略，不影响解析（向前兼容） |
| DATA_L1_003_必要字段缺失 | P0 | CC 数据中缺少必要字段（如 sessionId） | 跳过该条记录，不崩溃，继续处理下一条 |
| DATA_L1_004_完全无法解析 | P0 | CC 数据文件格式完全不可识别 | 功能降级（该模块数据不可用），其他功能正常 |
| DATA_L1_005_项目哈希生成 | P1 | 给定项目绝对路径 | project-hash = SHA-256(项目绝对路径).substring(0, 16)，哈希结果稳定一致 |

### 2.2 聊天历史同步

| 用例编号 | 优先级 | 场景 | 预期结果 |
|---------|--------|------|---------|
| DATA_L1_010_启动全量扫描 | P0 | 应用启动，~/.claude/ 下有 5 个会话 | 全量扫描并同步 5 个会话到 Muxvo 镜像 |
| DATA_L1_011_增量同步 | P0 | 运行中 CC 新增 1 个会话 | chokidar 检测到新文件，增量同步该会话 |
| DATA_L1_012_去重 | P0 | 同一 sessionId 重复出现 | 按 sessionId 去重，不产生重复记录 |
| DATA_L1_013_mtime精度 | P1 | 文件 mtime 有毫秒差异 | 使用 Math.floor(mtimeMs/1000) 秒级精度，毫秒差异不影响去重判断 |
| DATA_L1_014_仅同步不删除 | P0 | CC 端删除一个会话 | Muxvo 镜像中保留该会话，不跟随删除 |
| DATA_L1_015_同步状态记录 | P1 | 完成一次同步 | sync-state.json 更新，记录最新同步时间和状态 |
| DATA_L1_016_会话更新刷新 | P0 | 已同步的 session 在 CC 端追加内容 | 检测到文件变化，实时刷新该 session 数据 |

### 2.3 文件监听

| 用例编号 | 优先级 | 场景 | 预期结果 |
|---------|--------|------|---------|
| DATA_L1_020_新文件标记 | P1 | CC 端新创建会话文件 | 标记为 NEW，触发同步 |
| DATA_L1_021_忽略规则 | P1 | 目录中出现 .DS_Store / .swp / 以~结尾的文件 | chokidar 忽略这些文件，不触发同步 |
| DATA_L1_022_实时刷新 | P0 | session 文件被 CC 更新 | 界面实时刷新该 session 的内容 |

### 2.4 搜索索引

| 用例编号 | 优先级 | 场景 | 预期结果 |
|---------|--------|------|---------|
| DATA_L1_030_首次建索引 | P0 | 首次启动，有 100 个会话 | 后台 Web Worker 构建倒排索引 |
| DATA_L1_031_渐进可用 | P0 | 索引构建中 | 已索引的内容可立即搜索，未索引部分等待 |
| DATA_L1_032_增量更新 | P1 | 索引建好后新增会话 | chokidar 触发增量索引更新 |
| DATA_L1_033_索引持久化 | P1 | 关闭应用再重启 | 索引从持久化文件加载，无需完全重建 |
