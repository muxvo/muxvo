### 1.2 状态转换路径覆盖表

| 编号 | 起始状态 | 目标状态 | 触发条件 | 覆盖用例 |
|------|---------|---------|---------|---------|
| T1 | [*] | Launching | 用户双击应用图标 / 命令行启动 | APP_L1_001 |
| T2 | Launching | Restoring | 主进程初始化完成，读取 config.json | APP_L1_002 |
| T3 | Restoring | RestoringTerminals | config.openTerminals.length > 0 | APP_L1_003 |
| T4 | Restoring | EmptyState | config.openTerminals.length === 0 | APP_L1_004 |
| T5 | RestoringTerminals | Running | 所有终端恢复完成 | APP_L1_003 |
| T6 | EmptyState | Running | 空白界面渲染完成 | APP_L1_004 |
| T7 | Running | Saving | 窗口 close 事件 | APP_L1_005 |
| T8 | Saving | ShuttingDown | 状态保存完成 | APP_L1_005 |
| T9 | ShuttingDown | [*] | 所有子进程 exit | APP_L1_006 |

### 1.3 异常路径覆盖表

| 编号 | 场景 | 触发条件 | 预期行为 | 覆盖用例 |
|------|------|---------|---------|---------|
| E1 | config.json 损坏 | JSON 解析失败 | 使用默认配置启动 | APP_L2_001 |
| E2 | config.json 不存在 | 首次安装 | 触发 Onboarding 流程 | APP_L2_002 |
| E3 | 终端恢复部分失败 | PTY spawn 失败 | 标记失败终端为红色，其余正常恢复 | APP_L2_003 |
| E4 | 关闭时子进程无响应 | SIGTERM 超时 | 强制 SIGKILL | APP_L2_004 |

## 2. L1 用例（功能验证）

### 2.1 启动与恢复

| 用例编号 | 优先级 | 场景 | 预期结果 |
|---------|--------|------|---------|
| APP_L1_001_冷启动 | P0 | 首次安装后启动应用 | 状态 [*] → Launching → Restoring → EmptyState → Running；显示空白主界面；config.json 按默认值生成（1400x900, fontSize 14, theme dark） |
| APP_L1_002_配置读取 | P0 | 启动时读取 config.json | 正确读取窗口尺寸、字体大小、主题、openTerminals 列表 |
| APP_L1_003_会话恢复 | P0 | 上次关闭时有 3 个终端 | 恢复 Grid 布局 → 恢复 3 个终端 → 检测各终端前台进程状态 → 状态点颜色正确更新 |
| APP_L1_004_空会话启动 | P0 | 上次关闭时无终端 (openTerminals=[]) | Restoring → EmptyState → Running；显示空白主界面 |
| APP_L1_005_正常关闭 | P0 | 点击窗口关闭按钮 | Running → Saving（保存当前终端列表与布局到 config.json）→ ShuttingDown → [*] |
| APP_L1_006_子进程退出 | P0 | 关闭应用时有 2 个运行中终端 | 所有子进程收到退出信号并 exit 后，应用进程退出 |
| APP_L1_007_默认配置值 | P1 | 删除 config.json 后启动 | 窗口尺寸 1400x900，fontSize 14，theme dark |
| APP_L1_008_窗口尺寸持久化 | P1 | 调整窗口到 1600x1000 后关闭再启动 | 重启后窗口尺寸为 1600x1000 |
