# Muxvo 操作指南

> 本文档是 Muxvo 的完整操作说明。当用户询问 Muxvo 的使用方法时，请基于本文档内容回答。

## Muxvo 是什么

Muxvo 是一个为 AI CLI 工具（Claude Code、Codex、Gemini CLI 等）重度用户打造的桌面工作台。它把多项目终端管理、聊天历史浏览、文件预览、配置可视化整合在一个窗口中，替代在多个终端窗口之间来回切换的低效工作方式。

**核心理念**：终端永远是主角。打开 Muxvo 看到的是终端，其他功能（聊天历史、文件浏览、配置管理）都是"在终端旁边展开"的辅助面板。

---

## 界面布局

Muxvo 的界面从上到下分为三部分：

1. **菜单栏**（顶部 36px）：左侧是 Muxvo 标题 + 功能选项卡，右侧是设置按钮
2. **主内容区**：终端平铺网格，占据大部分屏幕
3. **浮动面板**：点击选项卡后弹出的全屏覆盖面板（聊天历史、Skills、MCP 等）

### 菜单栏选项卡

| 选项卡 | 功能 |
|--------|------|
| Terminals | 返回终端视图（关闭所有浮动面板） |
| Skills | 打开 Skills 管理面板，浏览/编辑本地 Skills |
| MCP | 打开 MCP 服务器配置面板 |
| Hooks | 打开 Hooks 管理面板 |
| Plugins | 打开 Plugins 管理面板 |
| Chat | 打开聊天历史浏览器 |

## 终端管理

### 新建终端

- 点击屏幕右下角的 **"+"** 浮动按钮
- 或使用快捷键 **⌘T**（macOS）
- 新建时会弹出目录选择器，选择工作目录后打开 shell
- 在设置中可配置"启动时自动创建的终端数量"（默认 1，最大 20）

### 两种视图模式

#### 平铺模式（Tiling）

这是默认模式。所有终端平铺在一个网格中，智能自适应数量：
- 1 个终端 = 全屏
- 2 个 = 左右对半
- 3 个 = 三列
- 4 个 = 四宫格
- 更多终端时自动计算最佳行列布局

**在平铺模式下可以：**
- **拖拽排序**：拖拽终端头部（标题栏）可以调换终端位置
- **调整大小**：拖拽终端之间的边界线可以调整各终端的大小比例；双击边界线恢复等分

#### 聚焦模式（Focused）

双击任一终端（或点击终端头部的放大按钮 ↗）进入聚焦模式：
- 被选中的终端放大到左侧（占 75% 宽度）
- 其他终端缩小排列在右侧栏（最多显示 3 个，可滚动）
- 点击右侧栏中的终端可以直接输入内容（单击激活，双击切换聚焦）
- 按 **Esc** 或点击头部的网格图标返回平铺模式

> **设置项**：可在设置中开启/关闭"双击聚焦"功能。关闭后双击不会进入聚焦模式，只能通过放大按钮进入。

### 终端头部（Tile Header）

每个终端顶部有一行头部信息栏，从左到右包含：

| 元素 | 说明 |
|------|------|
| 状态指示点 | 绿色 = 等待输入（空闲），蓝色动画 = 运行中，灰色 = 已关闭 |
| Worktree 徽章 | 如果终端在 git worktree 中，显示绿色项目名标签 |
| 工作目录路径 | 显示缩短的 cwd 路径，**可点击**切换目录 |
| 自定义名称 | 点击路径右侧的"命名"区域可给终端起名字，格式显示为 `~/path · 名称` |
| WaitingInput 徽章 | 当终端等待用户输入时显示数字提示 |
| Worktree 按钮 | 分支图标，点击打开 Worktree 管理弹窗（仅 git 仓库显示） |
| 文件按钮 | 文件夹图标，点击打开文件浏览面板 |
| 放大/返回按钮 | 放大图标进入聚焦模式 / 网格图标返回平铺 |
| 关闭按钮 | × 关闭终端 |

### 切换工作目录

点击终端头部的工作目录路径，弹出 **CwdPicker**（目录选择器），可以：
- 浏览文件系统选择新目录
- 切换后终端会自动 cd 到新目录

### 关闭终端

- 点击头部的 × 按钮
- 如果有前台进程正在运行，会提示是否终止

## Worktree 管理

在 git 仓库的终端中，点击头部的分支图标（🔀）打开 Worktree 弹窗：

- **查看现有 Worktrees**：列出当前仓库的所有 worktree，显示分支名和状态
- **新建 Worktree**：点击"+ New Worktree"按钮，自动创建新的 worktree 并在新终端中打开
- **打开已有 Worktree**：点击列表中的 worktree，在新终端中打开该 worktree 目录

Worktree 终端会在头部显示绿色的项目名标签，方便识别。

### 工作区管理

点击菜单栏或通过快捷操作打开工作区管理弹窗，可以：

- **保存工作区**：将当前所有终端的布局（位置、大小、工作目录、自定义名称）保存为一个工作区快照
- **恢复工作区**：从已保存的工作区列表中选择，一键恢复之前的终端布局
- **重命名工作区**：给保存的工作区起一个易于辨识的名称

适用于需要在不同项目或任务之间切换的场景，避免每次重新手动创建和排列终端。

## 聊天历史浏览器

点击菜单栏的 **Chat** 选项卡打开。采用三列邮件客户端式布局：

### 左列：项目和会话列表
- 显示所有使用过 CC/Codex 的项目
- 每个项目下按时间排列会话记录
- 会话标记来源：`claude-code` 或 `codex`

### 中列：对话内容
- 选中一个会话后，中列显示完整的对话消息流
- 支持 Markdown 渲染

### 操作功能
- **搜索**：跨项目、跨会话的全文搜索
- **导出**：将会话导出为文件
- **续聊**：选中历史会话后，可在终端中继续该对话

### 数据来源
聊天历史从以下位置读取（只读）：
- Claude Code：`~/.claude/projects/` 目录下的 session JSONL 文件
- Codex：`~/.codex/sessions/` 目录下的会话文件
- Muxvo 会自动镜像备份聊天记录到 `~/Library/Application Support/Muxvo/chat-history/`

## 文件浏览与预览

### 打开方式
- 点击终端头部的 **文件夹图标**，右侧滑出文件浏览面板
- 显示该终端工作目录下的文件树

### 文件临时视图（FileTempView）
点击文件后进入三列全屏视图：
- **左列**：关联的终端列表
- **中列**：文件内容预览
- **右列**：文件目录树

### 支持的文件格式
- **Markdown**：渲染预览
- **代码文件**：语法高亮显示
- **图片**：直接预览
- **PDF**：内置 PDF 阅读器
- **电子表格**：表格视图

### 文件监听
- Muxvo 会监听工作目录的文件变化
- 当 CC 创建新文件（如 plan.md）时，文件浏览器自动高亮标记 **NEW**

## Skills 管理

点击菜单栏的 **Skills** 选项卡打开。三列布局：

### 左列：Skills 列表
- 显示本地已安装的所有 Skills（来自 `~/.claude/skills/`）
- 支持聚合浏览更多 Skills（来自 Anthropic 官方、SkillsMP、GitHub 等）

### 中列：文件树
- 显示选中 Skill 的文件结构（SKILL.md、references/、scripts/ 等）

### 右列：编辑器
- 查看和编辑 Skill 文件内容

### Skill 安装
- 从聚合浏览器中选择 Skill，一键安装到 `~/.claude/skills/`
- 安装后自动刷新列表

### AI 评分（V2 功能）
- 对 SKILL.md 进行 6 维度 AI 评分：实用性、工程质量、意图清晰度、设计巧妙度、文档完善度、可复用性
- 生成雷达图评分卡 + 等级称号（Promising → Solid → Advanced → Expert → Masterwork）
- 评分卡可保存为 PNG 或复制分享

## MCP 配置

点击菜单栏的 **MCP** 选项卡打开。两列布局：

- **左列**：MCP 服务器列表（读取自 `~/.claude/mcp.json`）
- **右列**：选中服务器的配置详情

可查看和管理已配置的 MCP（Model Context Protocol）服务器。

---

## Hooks 管理

点击菜单栏的 **Hooks** 选项卡打开。两列布局：

- **左列**：Hooks 列表（读取自 `~/.claude/hooks/` 和 `settings.json`）
- **右列**：Hook 详情和源码查看

### 安全审查
安装新的 Hook 时，Muxvo 会弹出安全审查对话框，展示完整源码和权限声明，用户确认后才执行安装。因为 Hooks 可以执行任意代码，必须让用户知情。

---

## Plugins 管理

点击菜单栏的 **Plugins** 选项卡打开。两列布局：

- **左列**：已安装的插件列表（读取自 `~/.claude/plugins/config.json`）
- **右列**：插件配置详情

## 富编辑器

Muxvo 用 Web 富编辑器替代了终端原生输入，提供更好的文本编辑体验：

### 基本功能
- 支持鼠标点击定位光标
- 支持鼠标选中文本
- 支持多行编辑（自然换行，不需要记忆终端多行快捷键）
- 编辑完成后按 **Enter** 发送到终端

### 图片粘贴
- 直接 **Ctrl/⌘+V** 粘贴剪贴板中的图片
- 图片会保存为临时文件，发送时自动插入文件路径或模拟粘贴操作
- 支持 Claude Code 的图片输入协议

### 自动模式切换
- 当运行全屏 TUI 程序（vim、htop 等）时，自动切换到原始终端模式
- TUI 程序退出后自动恢复富编辑器模式

### 多行协议适配
- Claude Code：使用 `Escape+Enter` 换行协议
- bash/zsh：直接发送

## 设置

点击菜单栏右侧的 **齿轮图标** 或按 **⌘,** 打开设置弹窗。

### 通用设置
| 设置项 | 说明 |

### 语言
| 设置项 | 说明 |

|------|------|

## 数据与隐私

- Muxvo 所有核心功能在本地运行，不联网上报用户数据
- 聊天历史、配置文件均读取自本地文件系统
- AI 评分功能通过 CC Skill 在本地完成，Skill 内容不经过 Muxvo 服务器
- Showcase 展示页发布功能需要联网，但仅上传用户主动选择发布的内容

### 数据目录
- **Dev 模式**：`~/Library/Application Support/Muxvo Dev/`
- **正式版**：`~/Library/Application Support/Muxvo/`（macOS）

存储的数据包括：应用配置、用户偏好（主题/语言等）、聊天历史镜像、Skills 评分缓存等。

## 常见问题

### Q: 关闭 Muxvo 后终端里的进程会怎样？
A: Muxvo 关闭时，所有子进程会随之终止。重新打开 Muxvo 后会在之前的目录重新打开 shell，但之前的输出和上下文不会恢复。可以通过聊天历史浏览器回顾之前的对话。

### Q: Muxvo 最多能开多少个终端？
A: 最多 20 个终端。

### Q: 支持哪些 AI CLI 工具？
A: 终端管理完全通用，可以运行任何 CLI 工具。聊天历史和配置管理目前深度集成 Claude Code 和 Codex。

### Q: 怎么更新 Muxvo？
A: 在 设置 → 检查更新 中检查新版本。Muxvo 支持自动更新，检测到新版本后可一键下载安装。
