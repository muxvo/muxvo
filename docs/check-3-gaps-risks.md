## 5. 技术风险遗漏

### 5.1 DEV-PLAN 风险评估中已覆盖的风险

DEV-PLAN 第 18 节覆盖了 6 项外部依赖风险。但仅限外部依赖，遗漏了多个内部技术风险。

### 5.2 PRD 有但风险评估未覆盖的风险

| # | 风险描述 | 来源 | 影响 | 严重度 |
|---|---------|------|------|--------|
| 1 | **xterm.js 兼容性与版本锁定**：xterm.js 是 Muxvo 最核心的依赖，其 API 变更（如 FitAddon、WebLinksAddon 等插件的接口变化）可能导致大面积代码修改 | PRD 12.1 推荐 xterm.js | V1 全部终端功能 | 🔴 高 |
| 2 | **Electron 打包体积**：Electron + node-pty + xterm.js + chokidar + markdown-it + CodeMirror 6，最终安装包可能超过 200MB，影响用户下载意愿 | 整体架构 | 用户获取 | 🟡 中 |
| 3 | **node-pty 原生模块编译**：node-pty 需要原生编译（node-gyp），在不同 Node.js/Electron 版本下可能出现编译问题，尤其是 Apple Silicon 上的 arm64 兼容 | DEV-PLAN 11.1 | V1-P0 启动 | 🔴 高 |
| 4 | **contenteditable 跨浏览器行为不一致**：Chromium 版本间 contenteditable 的行为（换行、粘贴格式化文本、IME 输入法）可能不同，Electron 内嵌 Chromium 版本锁定但升级时需回归测试 | PRD 8.16 | V1-P1 富编辑器 | 🟡 中 |
| 5 | **CC 数据格式变更**：Claude Code 是活跃开发中的产品，history.jsonl、session JSONL、settings.json 的格式可能随 CC 版本升级而变更，导致 Muxvo 解析失败 | PRD 3.3 | V1 全部 CC 集成功能 | 🔴 高 |
| 6 | **tcgetpgrp 跨平台差异**：macOS 通过 `ps` 获取进程名 vs Linux 通过 `/proc/{pid}/comm`，Windows 完全不同——DEV-PLAN 仅描述了 macOS 方案 | PRD 6.7/附录 E | V1-P0（Windows/Linux 支持） | 🟡 中 |
| 7 | **大量终端的内存/CPU 压力**：20 个 xterm.js 实例 + 20 个 node-pty 子进程 + chokidar 监听 20 个目录，可能导致内存超过 2GB 限制 | PRD 11.2 | V1 性能 | 🟡 中 |
| 8 | **Alternate Screen Buffer 检测延迟**：在 ASB 信号和实际 UI 模式切换之间可能有短暂延迟，用户在 vim 启动瞬间可能仍在编辑器模式输入 | PRD 8.16 | V1-P2 富编辑器 | 🟢 低 |
| 9 | **搜索索引构建阻塞**：大量 session JSONL（>1000 个文件）首次建索引可能耗时过长（>5 分钟），影响首次使用体验 | PRD 11.2 | V1-P1 搜索 | 🟡 中 |
| 10 | **聊天历史镜像同步竞争条件**：CC 正在写入 session JSONL 时 Muxvo 同时读取可能导致读到不完整的 JSON 行 | PRD 11.2 | V1-P0 聊天历史 | 🟡 中 |
| 11 | **Electron 自动更新机制**：DEV-PLAN 未提及 Muxvo 自身的自动更新方案（electron-updater / Squirrel 等），这对产品分发至关重要 | - | 产品分发 | 🔴 高 |
