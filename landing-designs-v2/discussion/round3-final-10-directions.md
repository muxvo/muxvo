# 最终 10 个设计方向（投票共识 + 多样性覆盖）

> 三位 designer 的核心共识：**Amber Mission Control 融合方案**是首选。
> 但用户需要 10 个不同风格的方案供对比选择。因此我们在共识核心上做变体探索，同时保留几个差异化方向作为对照。

---

## A 组：核心共识方向（5 个变体）

### 01. Amber Mission Control — 标准版
融合三人共识的"最佳实践"版本。
- **amber 占比**: 15-18%（D2 建议的中间值）
- **布局**: 面板矩阵（CSS Grid），Hero 用产品截图 + amber 径向光晕
- **光影**: 滚动点亮效果（Cinematic Spotlight 技法）
- **文案调性**: "Your AI CLI Mission Control"
- **特色元素**: 面板卡片 + amber 上边框 + hover 内发光

### 02. Amber Signal Maximalist — 琥珀信号极致版
D3 主推的"amber 霸屏"策略，推到极致。
- **amber 占比**: 25-30%（D3 原始建议）
- **布局**: 交替 section（深底 amber 字 ↔ amber 底深字）
- **特色**: 至少 2 个 amber 反转色带 section，大面积径向渐变背景
- **风险测试**: 验证高 amber 浓度是否真的更有记忆点

### 03. Mission Control + Blueprint — 蓝图指挥台
D1 的蓝图概念融入指挥中心框架。
- **amber 占比**: 12-15%
- **布局**: 面板矩阵 + 背景坐标网格（极低透明度 amber 网格线）
- **特色**: 标注线从面板拉出功能说明（D2 建议的"功能性标注"）
- **氛围**: 精密、工程化、技术制图感

### 04. Mission Control + Terminal Hero — 终端入场版
D3/D1 建议的"CLI 输入 → GUI 展开"微交互。
- **amber 占比**: 15-18%
- **Hero 特殊处理**: 先展示 `$ muxvo launch` 终端命令行，回车后展开为产品截图
- **正文区域**: 回归标准面板矩阵布局
- **特色**: Hero 区的 CLI→GUI 转场传达产品核心价值

### 05. Amber Cinematic — 暗室聚光版
D1+D2 推荐的电影光影路线，amber 做唯一光源。
- **amber 占比**: 18-22%
- **布局**: 非面板矩阵，而是全屏 section 逐个点亮
- **光影**: 每个 section 独立的 amber 径向光晕，滚动时渐进点亮
- **氛围**: 沉浸、戏剧、高级感
- **与 01 区别**: 01 偏信息密度，05 偏叙事沉浸

## B 组：差异化探索方向（3 个）

### 06. Orchestrator — 编排者
D3 提案，节点连线展示多 CLI 编排价值。
- **amber 占比**: 12-15%
- **Hero**: 产品截图 + 定位文案（非连线图）
- **第二屏**: 核心价值区用节点连线图（Claude Code/Codex/Gemini CLI logo 节点 + amber 曲线连线 + 光点流动动画）
- **功能区**: 每个功能是连线图中的一个展开面板
- **差异点**: 唯一明确传达"连接多工具"价值的方案

### 07. Terminal Native — 终端原生
全站终端视觉语言版本，作为风格对照组。
- **amber 占比**: 10-12%（amber 替代传统终端绿）
- **全站 monospace 字体**，ASCII art logo
- **命令行风格导航**，tree 目录功能展示
- **"CLI 输入 → GUI 输出"反差**: 命令输入区终端风格，输出结果用现代卡片
- **差异点**: 目标用户身份认同感最强

### 08. Split Persona — 双面对比叙事
Before/After 对比作为全站叙事线索。
- **amber 占比**: 15-18%（仅 After 侧有 amber）
- **Hero**: 分屏设计，左灰右暖，amber 分割线
- **每个功能区**: 都有 mini 的 before/after 对比
- **特色**: 痛点驱动的叙事方式，先展示问题再给方案
- **差异点**: 唯一以"用户痛点"为叙事起点的方案

## C 组：风格边界探索（2 个）

### 09. Craft Object — 精工器物
工匠质感路线，完全不同的视觉气质。
- **amber 用法**: 黄铜配件式（小面积、高质感金属渐变）
- **背景**: 带微妙深色纹理（非纯平色）
- **边框**: 1px 实线 + 极微弱内阴影，无圆角
- **字重**: 标题 700 / 正文 400 强对比
- **无任何 glow/blur 效果**: 每个元素都是"实体"
- **差异点**: 完全跳出科技/SaaS 审美的工匠路线

### 10. Amber Signal + Docs — 文档信号塔
D2 的"文档即官网"理念 + D3 的 amber 色彩策略。
- **amber 占比**: 20-25%
- **布局**: 左侧固定导航 + 右侧内容区（文档站风格）
- **amber 用法**: 导航高亮、代码块 accent、section 标题装饰
- **内容**: 官网即快速入门指南，代码块 + 安装步骤 + 功能文档
- **差异点**: 最务实、最"开发者友好"的方案

---

## 分组逻辑
- **A 组 (01-05)**: 核心共识的 5 种浓度/侧重变体，验证"最佳 amber 浓度"和"最佳布局"
- **B 组 (06-08)**: 讨论中有支持者的差异化方向，提供真正不同的选择
- **C 组 (09-10)**: 边界探索，看看完全不同的方向能否带来惊喜
