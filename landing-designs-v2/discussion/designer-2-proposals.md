# Designer-2 风格提案（用户体验实用派视角）

> 核心立场：开发者用户的注意力极其稀缺。官网的首要任务不是"好看"，而是在 3 秒内让用户理解"这个工具解决了我什么问题"，然后给出一个无摩擦的下载路径。一切视觉设计都应服务于这个目标。

---

## 提案 1：工具台面风格 / Workbench Utility

### 为什么适合 Muxvo
Muxvo 的核心隐喻就是"工作台"——多个终端平铺、工具随手可取。官网应该延续这个隐喻，让用户一进来就感受到"这是一个为效率而生的工具"。不需要抽象概念，直接展示产品界面本身就是最好的卖点。开发者最讨厌的就是看了半天官网还不知道产品长什么样。

### 视觉关键词
实用、克制、信息密度高、工具感、网格布局、功能即装饰、零废话

### 配色思路
- 主背景：#0a0e14 → #161b22 的深色梯度，与产品 UI 完全一致
- Amber #e8a748 仅用于关键 CTA 按钮和核心数据高亮（下载量、功能数字等）
- 辅助色极度克制：仅用 --text-secondary (#71717a) 做次要文字
- 不引入品牌色以外的任何装饰色
- 整体色彩占比：深色 85% / 中性灰 10% / Amber 5%

### 布局核心策略
- Hero 区域：左侧一句话标题 + 下载按钮，右侧直接放产品截图（真实 UI，不是 mockup）
- 功能区：采用 Muxvo 本身的"平铺网格"布局展示 4-6 个核心功能，每个格子 = 小截图 + 一句话
- 没有 parallax、没有滚动动画、没有渐变 blob——加载即完整呈现

### 参考产品/网站
- **Warp Terminal 官网**（warp.dev）——产品截图即 hero，功能分区清晰
- **Raycast 官网**（raycast.com）——信息密度高但不拥挤，CTA 路径极短

### 风险/挑战
- 如果产品截图质量不够高（分辨率、内容填充），会显得寒酸而非"工具感"
- 信息密度高的设计在移动端需要重新编排，响应式工作量较大
- 可能在设计审美上显得"朴素"，缺乏第一眼的视觉冲击力

## 提案 2：暗室聚光风格 / Dark Spotlight

### 为什么适合 Muxvo
AI CLI 工具的用户几乎 100% 使用深色主题。官网用近黑背景不是为了酷，而是为了让用户感到"这是我的同类"。在纯暗背景下，用 Amber 光源效果聚焦每个功能模块，形成"暗室中的聚光灯"效果——每次只让用户关注一个信息点，降低认知负荷。

### 视觉关键词
深黑、聚焦、光晕引导、逐步揭示、对比度极高、戏剧感、单点照亮

### 配色思路
- 背景：纯 #06080c 到 #0a0e14，比产品 UI 还要深一级
- Amber 作为"光源"存在：功能卡片底部有柔和的 amber glow（类似 --accent-glow 但更大范围）
- 文字：标题用 #ffffff 纯白（在极暗背景下获得最大对比度），正文用 #d4d4d8
- 每个 section 之间用"光→暗→光"的节奏形成视觉呼吸

### 布局核心策略
- Hero：全黑背景，中央一个产品截图从暗到亮"点亮"（CSS 动画，不依赖 JS）
- 向下滚动时，每个功能区域在进入视口时"亮起"，离开时"暗下去"
- 底部 CTA 区域用最强烈的 amber glow，形成"终点引力"

### 参考产品/网站
- **Linear 官网**（linear.app）——极暗背景 + 精确光影 + 高级感
- **Vercel 官网**（vercel.com）——暗色调中用光效引导视觉焦点

### 风险/挑战
- 聚光效果如果做过了会变成"游戏官网"，需要严格控制光晕半径和透明度
- 纯暗背景对内容质量要求极高，任何粗糙的图片或文案都会被放大
- Intersection Observer 驱动的亮暗动画需要注意性能，低端设备可能卡顿

## 提案 3：终端原生风格 / Terminal Native

### 为什么适合 Muxvo
Muxvo 服务的是终端重度用户。与其用常规网页设计去"翻译"产品价值，不如直接用终端的视觉语言来表达——monospace 字体、命令行提示符、ASCII 风格边框。这种设计会让目标用户产生强烈的身份认同："这个产品懂我"。同时，终端风格天然适合展示 CLI 命令和代码片段。

### 视觉关键词
Monospace、命令行、光标闪烁、ASCII art、黑绿/黑 amber、Hacker 美学、代码即设计

### 配色思路
- 背景：#0a0e14（与 Muxvo 终端背景一致）
- 主文字：#d4d4d8，但标题和关键信息用 Amber #e8a748 模拟"终端高亮"
- 命令提示符 `$` / `>` 用 Amber，命令输出用标准灰
- 可以引入少量 --success (#4ade80) 作为"命令执行成功"的视觉隐喻
- 边框用 ASCII 字符（`+---+`、`|   |`）或极细的 1px #1e2530

### 布局核心策略
- Hero：模拟一个终端窗口，里面"打出" Muxvo 的介绍文字（打字机效果）
  - `$ muxvo --what "AI CLI workbench for power users"`
  - `$ muxvo --features "tiling terminals, chat history, skill market"`
  - `$ muxvo install` → 下载按钮
- 功能区：每个功能用一个"终端面板"展示，左侧是命令/代码，右侧是效果截图
- 整个页面像一个大型终端 session 的输出记录

### 参考产品/网站
- **Fig（现 Amazon Q）早期官网**——用终端动画展示产品能力
- **Charm.sh**（charm.sh）——终端工具用终端美学做官网的经典案例

### 风险/挑战
- 打字机效果如果太慢会考验用户耐心，必须可跳过或极快（< 2秒完成）
- 纯终端风格可能劝退非硬核用户，限制产品的受众天花板
- Monospace 字体在长段落中可读性不如 sans-serif，功能描述需要精简到极致
- 这种风格容易做得"像玩具"，需要用真实的产品内容来撑住

## 提案 4：数据驱动信任风格 / Data-Driven Trust

### 为什么适合 Muxvo
开发者工具市场已经充斥着"好看但不知道做什么"的官网。Muxvo 的差异化在于具体的功能集——平铺终端、聊天历史搜索、Skill 市场 + AI 评分。与其用华丽的动画包装，不如用具体的数字和对比来建立信任：安装后省了多少时间？支持多少个 CLI 工具？Skill 市场有多少个包？让数据替产品说话。

### 视觉关键词
数字突出、对比表格、指标卡片、信任徽章、证据优先、无形容词、结果导向

### 配色思路
- 背景：#0d1117 → #161b22（比纯黑亮一级，增加"数据面板"的可信感）
- 数字/指标用大号 Amber #e8a748（48px-72px 的数字字号）
- 对比表格用交替行高亮：#0d1117 / #161b22
- 成功指标用 --success (#4ade80)，Muxvo 品牌特色用 Amber
- 引入微量 --info (#60a5fa) 作为链接色和次要信息高亮

### 布局核心策略
- Hero：不用形容词，直接三个大数字
  - "3 AI CLIs, 1 Workbench" 或 "管理 N 个终端 / 搜索 N 条聊天 / N 个 Skill 包"
  - 下方紧跟下载按钮
- 对比区：Muxvo vs 裸终端的效率对比（两栏表格，左边是痛点，右边是 Muxvo 解决方案）
- 功能区：每个功能 = 指标数字 + 一句话 + 小截图，不超过 3 行
- 社区证据区：GitHub stars、下载量、用户评价（如果有的话）

### 参考产品/网站
- **Supabase 官网**（supabase.com）——开源项目用数据和对比建立信任
- **PlanetScale 官网**——性能数字作为核心说服力

### 风险/挑战
- 新产品可能没有足够的数据来填充（下载量、用户数还不够大）
- 如果数字不够 impressive，反而会暴露产品的早期阶段
- 需要持续维护数据的准确性，静态页面上写死数字容易过时
- 过度强调数据可能让页面显得冰冷，缺乏情感连接

## 提案 5：渐进揭示风格 / Progressive Reveal

### 为什么适合 Muxvo
Muxvo 的功能很多（终端管理、聊天历史、文件浏览、配置编辑、Skill 市场），一次全部展示会让用户信息过载。渐进揭示（Progressive Disclosure）是 UX 的核心原则——先给用户最关键的信息，感兴趣了再深入。这种设计特别适合功能丰富但需要"先理解核心再了解全貌"的工具。

### 视觉关键词
层次分明、折叠展开、先简后繁、呼吸感、空白大方、重点突出、节奏清晰

### 配色思路
- 背景分层：Hero 用 #06080c（最深），每个 section 递增亮度到 #161b22
- 这种"从深到浅"的渐变暗示"逐步揭开"的信息层级
- Amber #e8a748 作为"当前激活层"的标识色
- 大量使用 #71717a 次要文字色来降低非核心信息的视觉权重
- Section 之间用渐变分隔，不用硬边线

### 布局核心策略
- Hero：只有一句话 + 一个按钮 + 一张图。例如："The workbench your AI CLI deserves." + [Download] + 产品截图
- 第二屏：核心功能概览（3 个 icon + 标题，每个可点击展开详情）
- 第三屏：选中功能的深度展示（大截图 + 详细说明）
- 第四屏：Skill 市场专区（这是差异化功能，值得单独一屏）
- 第五屏：快速开始 + 下载

### 参考产品/网站
- **Notion 官网**（notion.so）——层次分明，先简后繁，每个section只讲一件事
- **Arc 浏览器官网**（arc.net）——用最少的信息量建立兴趣，然后逐步深入

### 风险/挑战
- 渐进揭示意味着核心内容可能在"折叠"之后，如果用户不点开就流失了
- 需要精准判断"第一层放什么"——放错了等于没有渐进揭示
- 交互设计工作量比纯静态页面大，展开/折叠的动画需要打磨
- 移动端的"渐进揭示"容易变成"无限滚动"，需要另外设计移动端策略

## 提案 6：开发者文档风格 / Docs-First Landing

### 为什么适合 Muxvo
这是一个反直觉但可能非常有效的方向：把官网做成"增强版文档首页"。开发者在选择工具时，最终决策依据往往不是官网的视觉效果，而是文档的质量和清晰度。如果官网本身就是一份结构清晰的"快速入门指南"，用户在了解产品的同时就已经完成了 onboarding 的第一步。这种设计传达的信号是："我们是认真做工具的，不是做营销的。"

### 视觉关键词
文档感、左侧导航、代码块、有序列表、锚点定位、可搜索、内容为王

### 配色思路
- 背景：#0d1117（GitHub dark mode 同色系，开发者最熟悉的阅读环境）
- 代码块背景：#161b22，与正文形成层次
- Amber #e8a748 用于：侧边栏当前项高亮、重要标注、inline code 的 accent
- 链接色：#60a5fa（与 GitHub/docs 站一致，降低认知负担）
- 标题层级用字号和 Amber 粗细区分，不用颜色区分

### 布局核心策略
- 顶部：产品名 + 一句话描述 + [Download] + [GitHub] 两个按钮
- 左侧固定导航：Quick Start / Features / Installation / CLI Support / Skill Market / FAQ
- 右侧内容区：
  - Quick Start：3 步安装流程（带代码块）
  - Features：功能列表，每个带截图和代码示例
  - 底部：GitHub 链接、社区链接、changelog
- 整个页面可以 Cmd+F 搜索，信息完全平铺

### 参考产品/网站
- **Homebrew 官网**（brew.sh）——极简，直接就是用法
- **Tailwind CSS 官网**（tailwindcss.com）——文档即官网，内容质量驱动转化

### 风险/挑战
- 对于不了解 AI CLI 的潜在新用户，文档风格可能显得"门槛高"
- 缺乏情感化设计，难以在社交媒体传播（没人会截图分享一个文档页面）
- 左侧导航在移动端需要转为汉堡菜单或底部 tab，体验差异大
- 如果文档内容本身不够好，这种风格会直接暴露内容短板

## 综合建议（Designer-2 视角）

从用户体验实用派的角度，我的优先级排序：

| 排名 | 提案 | 理由 |
|------|------|------|
| 1 | 提案 1：工具台面 | 最直接、最低风险、开发成本最低，产品截图就是最好的说服力 |
| 2 | 提案 5：渐进揭示 | 信息架构最合理，适合功能丰富的产品，但交互成本较高 |
| 3 | 提案 2：暗室聚光 | 视觉高级感强，但需要优秀的截图素材支撑 |
| 4 | 提案 3：终端原生 | 身份认同感最强，但受众天花板最低 |
| 5 | 提案 6：文档优先 | 最务实，但缺乏传播性 |
| 6 | 提案 4：数据驱动 | 理论上最有说服力，但新产品阶段数据不足以支撑 |

**我最想组合的方向**：提案 1 的"产品截图即 hero" + 提案 2 的"amber 聚光效果" + 提案 5 的"信息层级节奏"。这个组合能兼顾：3 秒理解产品（截图）、视觉高级感（光效）、信息不过载（渐进）。
