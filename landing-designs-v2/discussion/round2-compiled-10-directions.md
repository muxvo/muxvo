# 第二轮讨论：10 个候选风格方向（汇总稿）

> 来源：3 位 designer 共 18 个提案，经合并同类项后精选 10 个方向。
> 请各位投票 + 补充意见，确认最终 10 个方向。

---

## 1. Mission Control / 任务指挥中心
**来源**: D3-提案1 | **共识度**: ★★★★★（三人均认可"指挥中心"定位）

- 核心概念：航天控制室隐喻，多面板矩阵布局，信息密度高
- 配色：深黑 + amber 状态灯 + 少量状态色（蓝绿=就绪）
- 关键元素：扫描网格、LED 数字、面板化功能区、实时感
- 与竞品差异：直接定义"指挥系统"新品类，避免被归为终端/IDE

## 2. Amber Signal / 琥珀信号塔
**来源**: D3-提案2 | **共识度**: ★★★★★（D3 首选，D1 认可 amber 作为差异武器）

- 核心概念：Amber 从配角升级为主角，占页面视觉面积 25-30%
- 配色：amber 径向渐变背景、amber 色带 section、暗底暖光
- 关键元素：大面积 amber 渐变光、amber 底 + 暗字的交替 section、强 CTA
- 与竞品差异：在冷色竞品海洋中做唯一的"暖色异类"

## 3. Blueprint Grid / 工程蓝图
**来源**: D1-提案2 | **共识度**: ★★★★☆（D1 首选）

- 核心概念：平铺布局升华为工程图纸，坐标网格 + 标注线
- 配色：深蓝黑 + amber 低透明度网格 + 标注线
- 关键元素：坐标纸网格、标注箭头、虚线框、精密刻度
- 与竞品差异：几乎无竞品用此风格，辨识度极高

## 4. Cinematic Spotlight / 电影聚光
**来源**: D1-提案1 + D2-提案2 融合 | **共识度**: ★★★★☆

- 核心概念：极暗背景 + amber 光源聚焦，每次只照亮一个信息点
- 配色：近黑背景 + amber 径向光晕 + 冷蓝反差光（极少量）
- 关键元素：光晕衰减、暗角、HUD 叠加、滚动触发亮/暗节奏
- 参考：Linear 的光影处理 + Vercel 的焦点引导

## 5. Terminal Native / 终端原生
**来源**: D1-提案6 + D2-提案3 融合 | **共识度**: ★★★★☆（目标用户共鸣最强）

- 核心概念：官网即终端 session，命令行视觉语言讲故事
- 配色：纯黑 + amber 提示符 + 灰色输出 + ASCII 边框
- 关键元素：打字机动画、命令行导航、tree 目录、光标闪烁
- 关键创新：CLI 输入 → GUI 输出的反差，传达 Muxvo 价值

## 6. Workbench Utility / 工具台面
**来源**: D2-提案1 | **共识度**: ★★★☆☆（D2 首选，最务实方向）

- 核心概念：产品截图即 Hero，零废话，3 秒理解产品
- 配色：与产品 UI 完全一致的深色调，amber 仅用于 CTA
- 关键元素：真实 UI 截图、高信息密度网格、无装饰动画
- 特点：开发成本最低，信息传达效率最高

## 7. Orchestrator / 编排者
**来源**: D3-提案3 | **共识度**: ★★★☆☆

- 核心概念：节点连线图传达"编排多工具"的核心价值
- 配色：暗底 + 点阵画布 + amber 连线 + 工具差异色图标
- 关键元素：曲线连线、数据流动动画、节点卡片、全局视角
- 与竞品差异：传达"不做具体事，编排做事的工具"

## 8. Split Persona / 双面对比
**来源**: D3-提案4 | **共识度**: ★★★☆☆

- 核心概念：Before/After 分屏叙事，痛点 → 解决方案
- 配色：Before 侧冷灰无 amber → After 侧 amber 暖光
- 关键元素：分屏布局、分割线渐变光、对比动画
- 风险：需要真实痛点场景，避免"电视购物感"

## 9. Craft Object / 精工器物
**来源**: D3-提案5 | **共识度**: ★★★☆☆（差异化最强但执行难度高）

- 核心概念：工匠质感路线，重量感、材质感、细节控
- 配色：深色带微妙纹理 + amber 做"黄铜配件"
- 关键元素：金属渐变、精细边框、内阴影、字重对比
- 与竞品差异：在科技/未来的方向中做唯一的"匠心"路线

## 10. Spatial Depth / 空间纵深
**来源**: D1-提案5 | **共识度**: ★★★☆☆

- 核心概念：3D 空间传达多面板协作，Z 轴层叠
- 配色：远深近浅渐变 + amber 边框光随层级增强
- 关键元素：透视旋转、卡片层叠、滚动视差、3D transforms
- 特点：现代感强，但需要精细的性能优化

---

## 淘汰说明
以下提案因重叠或阶段不适合被合并/淘汰：
- **Molten Amber** → 精华融入 #2 Amber Signal 和 #4 Cinematic Spotlight
- **Typographic Brutalism** → 中文大字排版风险高，精华融入 #9 Craft Object
- **Data-Driven Trust** → 产品早期数据不足，不适合单独成方案
- **Progressive Reveal** → 信息架构策略而非视觉风格，可融入任何方案
- **Docs-First** → 缺乏传播性，更适合做子页面
- **Living Dashboard** → 需要真实数据 API，适合产品成熟期
