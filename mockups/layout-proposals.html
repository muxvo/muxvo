文档内容
优势
  • 终端输出始终可见，方便参考构建日志/错误信息
  • 从终端错误定位文件 → 中间选文件 → 右侧看代码，流程自然
  • 视线从左到右：上下文 → 导航 → 内容，符合阅读习惯
考虑
  • 终端占据了左侧黄金位置，文件树被挤到中间
  • 如果终端不需要时，左侧空间浪费
  • 与 VS Code 等主流编辑器的布局习惯不同

优势
  • 与 VS Code / JetBrains 等主流 IDE 布局一致，零学习成本
  • 文件树在左侧是最自然的导航位置
  • 代码编辑区在中间获得最大空间和视觉焦点
  • 终端在右侧作为辅助参考，不抢主体注意力
考虑
  • 与当前 Muxvo 的实现完全一致（文件树 | 内容 | 终端）
  • 终端在右侧宽度受限，长命令输出可能需要横向滚动
